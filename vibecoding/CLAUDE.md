# vibecoding ドキュメント管理ルール

## MD → HTML 変換

- vibecoding配下のハンズオンドキュメント（ケバブケースの `.md` ファイル）を編集したら、必ずHTML変換を実行する
- `CLAUDE.md` など管理用ファイルは対象外
- コマンド: `python3 ~/.claude/scripts/md2html.py <入力.md> <出力.html>`
- ユーザーからの指示がなくても自動で行う。確認不要
- Git へのコミット・プッシュは明示的な指示があるときのみ行う

## 文体

- 基本的にである調で書く
- 「〜だ。」で終わる場合は体言止めにする
  - 例: 「シンプルな構成だ。」→「シンプルな構成。」
- ただしユーザーが「〜だ。」に修正することがあるので、その場合はそのまま従う

## 太字・バッククオートの使い方

- **太字** (`**`): UIのラベル（ボタン名、メニュー名など）
  - 例: **Deploy** をクリック、**New session** を選択
- バッククオート (`` ` ``): コマンド、コードスニペット、ファイル名・パス
  - 例: `wrangler pages deploy ./public`、`index.html`、`npm install`
- 固有名詞・サービス名はバッククオート不要
  - 例: Cloudflare Pages、GitHub Actions、localStorage、IndexedDB

## リビルド

「リビルドして」と言われたら以下を順番に実行する。

### 1. mdフッタの最終更新日を更新

各ケバブケースmdファイルについて、ファイルの更新日（`stat -f "%Sm" -t "%Y-%m-%d" <file>`）を取得し、フッタの日付と比較する。

- 初回公開日のみの場合（例: `2026-05-04`）→ `2026-05-04 (last updated: YYYY-MM-DD)` に変更
- すでに `last updated` がある場合 → その日付を更新
- ファイルの更新日とフッタの日付が一致している場合はスルー

### 2. 古いHTMLを再変換

mdファイルがHTMLより新しい場合、`python3 ~/.claude/scripts/md2html.py` でHTML変換する。

### 3. index.htmlの日付・タイトルを更新

index.htmlに掲載済みの各記事について以下を確認し、古ければ更新する。
- `<div class="date">` の日付
- リンクテキストがmdのh1と一致しているか
index.htmlに載っていないファイルはスルー。
